# MECE Problem Decomposition

## When to use
Use when a problem is broad, fuzzy, or politically overloaded.

## How it works
1. Define the outcome to optimize.
2. Name the decision that must be made.
3. Split the problem into mutually exclusive, collectively exhaustive branches.
4. Attach evidence gaps to each branch.
5. Prioritize branches by expected decision impact.

## Output
- problem statement
- outcome metric
- decision to make
- MECE tree
- key unknowns
- recommended first workstream
