# Rule: Cite the Method

**Always load.**

When the harness uses a named method, it names the method. When no named method fits, it says so.

## Rule

Program-management craft has a canon: Minto's Pyramid, Ishikawa's fishbone, Six Sigma's DMAIC, RACI, SCR, Mendelow's grid, Kepner-Tregoe decision analysis, and so on. The harness must cite the method it is applying.

This matters because:

1. **The user is signing an artifact.** An executive who asks "why are we structured this way?" needs an answer other than "the AI chose."
2. **The method comes with a Reviewer Checklist.** Named methods have known failure modes and known repairs; citing them activates that reviewer discipline.
3. **Composition requires vocabulary.** Agents compose skills via named methods, not via freestyle prompts.

## Application

- Using a fishbone diagram? Say "Ishikawa diagram (fishbone)."
- Using a power/interest grid? Say "Mendelow power/interest grid."
- Writing in BLUF / Pyramid? Say "Pyramid Principle (Minto)."
- Applying 5 Whys? Name it.
- Running DMAIC? Name the phase explicitly.

## When no named method fits

Say so explicitly: "No standard method applies; the structure below is custom because [reason]." Do not invent a method name.

## What the harness must not do

- **Invent method names** ("the BHAG framework" applied randomly).
- **Skip attribution** ("Here's a decomposition:" without naming that it's MECE by process-axis).
- **Confuse methods** (calling a RACI a power/interest grid; calling an SCR a Pyramid).

## Counter-examples the harness should reject

- "Here's a framework for thinking about this:" — what framework?
- "I've broken it down into four categories:" — by what axis?
- "Using the pyramid approach with five points on the second level" — Minto's Pyramid uses 3 at level 2 (per *The Pyramid Principle*, grouping rule). Five suggests the author didn't apply Minto at all.

## Source

- Craft discipline — methods have owners, citations, and known failure modes; citing them invokes all three.
- Academic honesty norms, ported to practitioner work.
