# Rule: Pyramid First

**Always load. No exceptions.**

Every output leads with the answer. No exceptions — including in chat replies, agent handoffs, brief openings, and memo structure.

## Rule

If the first paragraph of any output is context-setting, history, or "here's what I did," it is wrong. Restructure.

The answer comes first. The shape:

1. **Line 1** — the recommendation, decision, or BLUF
2. **Lines 2-4** — three supporting points that prove line 1
3. **Rest** — evidence, context, appendix

This is the Pyramid Principle (Barbara Minto, 1960s, McKinsey). It is not a stylistic preference; it is the contract the harness holds with a busy reader.

## Application

- Exec briefs: BLUF on line 1.
- Chat replies to the user: the answer to their question comes before any reasoning.
- Memos: the recommendation comes before the analysis that led to it.
- Post-mortems: the summary paragraph (what happened, what changed) is the opening, not the timeline.
- RAID rows: the state (e.g., severity 4, owner Marcus) is visible before the narrative.
- Status updates: the Resolution of SCR first; Situation and Complication follow.

## When it conflicts with narrative flow

It doesn't. Narrative that starts with the answer reads like a competent memo. Narrative that buries the answer reads like an undergraduate essay. Pick competent memo.

## Counter-examples the harness should reject

- "Over the past week, the team has been working on several fronts..."
- "As you know, we've been tracking the Japan situation..."
- "There are a number of considerations to think through..."

All three bury the lede. Rewrite.

## Source

- Barbara Minto, *The Pyramid Principle*, Chapter 1.
