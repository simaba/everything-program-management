# Rule: One Page or It's Not an Exec Brief

**Always load when executive-summary-brief skill is active.**

An executive brief is, by contract, one page. If the harness produces two pages, the brief has failed its contract.

## Rule

Hard cap: ~300–400 words. ~one printed page. No appendix in the brief; appendix is a separate, linked document.

## Why

Executive attention is the scarcest resource in the production of decisions. The brief format trades comprehensiveness for actionability. If you need two pages, the document is a memo, not a brief — and should be labeled that way.

## Application

- Cut from the bottom up. Background, history, "as you know" — first to go.
- Cut adverbs and hedges. "We may potentially want to consider..." → "Recommend X."
- Cut subordinate clauses. "Recognizing that, given the constraints we face..." → cut.
- If three supporting points won't fit, you have the wrong three. Pick the load-bearing three.

## Counter-examples the harness should reject

- A 1.5-page brief — round down to 1, not up to 2.
- "Brief with appendix attached" where the appendix is in the same file — split.
- Brief padded with sections like "Background" or "About this brief" — both are buried-lede signals; cut.

## What to do when one page is genuinely insufficient

You're writing a memo, not a brief. Switch artifacts:

- Use `decision-memo` if the goal is a decision with options.
- Use `scr-narrative` if the goal is a status update with structured framing.
- Use a long-form report (no skill in this repo for that — point the user to `templates/`).

The brief format is a contract; switching artifacts when the contract doesn't fit is the right move. Stretching the brief past one page is not.
