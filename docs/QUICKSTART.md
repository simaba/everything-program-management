# Quickstart

This repository helps you produce structured program-management artifacts quickly and consistently.

## Start in five steps

1. **Pick the artifact you need**
   - charter
   - RAID update
   - stakeholder map
   - decision memo
   - executive brief

2. **Choose the nearest entry point**
   - use `commands/` if you want a direct prompt shape
   - use `skills/` if you want the method itself
   - use `agents/` if you want a specialist role

3. **Draft against a template**
   Use the relevant file under `templates/` to keep output shape consistent.

4. **Check the example**
   Compare your draft with the examples under `examples/` before sending.

5. **Review before publication**
   Confirm that the output:
   - leads with the answer
   - names owners and dates
   - separates facts from assumptions
   - makes risks and open questions explicit

## Recommended starting points

| Need | Best starting point |
|---|---|
| New initiative kickoff | `commands/charter.md` |
| Weekly risk review | `agents/risk-triage.md` |
| Leadership one-pager | `agents/executive-summary.md` |
| Important decision with options | `skills/decision-memo/SKILL.md` |
| Stakeholder alignment plan | `skills/stakeholder-mapping/SKILL.md` |

## Ground rules

- prefer one clarifying question over three
- do not fake facts to complete a template
- mark missing information clearly
- keep executive outputs short and decision-oriented
